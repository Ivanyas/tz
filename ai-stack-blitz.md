# Мой AI-стек

## IDE и плагины

Работаю в VS Code, иногда переключаюсь на Cursor когда нужно закинуть в контекст много файлов сразу.

Из AI-плагинов:
- Copilot — для автодополнения, он быстрый
- Claude Code в терминале — когда нужно что-то сложное порефакторить или разобраться в чужом коде

Copilot хорош когда просто дописываешь строку, а Claude лучше думает когда нужно понять архитектуру или найти баг.

## Модели

В основном Claude Opus — он лучше держит контекст и реже выдумывает несуществующие методы. Для простых вещей типа "напиши коммент к функции" хватает Sonnet, он дешевле.

Copilot использует GPT-4, но там особо не выбираешь.

Когда нужно что-то загуглить — Claude с веб-поиском, чтобы не получить устаревшую инфу.

## MCP

Подключил пару MCP-серверов:
- filesystem — чтобы Claude сам мог читать файлы
- git — для статуса и диффов
- postgres — иногда удобно спросить "почему этот запрос тормозит" и он сам смотрит в схему

## Cursorrules

Да, использую. Без них Claude постоянно предлагает не тот стек или пишет код в другом стиле.

Основное что прописал:
- стек проекта (NestJS, TypeScript, TypeORM) — чтобы не предлагал Express или Prisma
- async/await везде, никаких callbacks
- типизация обязательна, any запрещён
- бизнес-логика в сервисах, запросы через репозитории

Ещё запретил создавать новые файлы без спроса и менять тесты — а то он любит "улучшить" то что работает.
